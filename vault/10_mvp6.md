## Link / Lane IDs

See `13_road_topology.md`.

- **Link ID** — inherited from MVP3 (Strategy A polygons).
- **Lane ID** — populated via **Strategy A extended**: the per-scene JSON
  gains lane strips `{ link_id, lane_id, polygon }`. The application-layer
  `RoadGraphAssigner` runs a second point-in-polygon pass to populate
  `VehicleState.lane_id`.

This is the first MVP where:

- **SSAM Lane-Change conflict classification works**
- **Per-lane analytics become meaningful**
